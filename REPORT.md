# Git + GitHub Lab Report

## 1. Что такое Git

Git — это распределённая система контроля версий, которая позволяет отслеживать изменения в проекте, хранить историю коммитов и работать с ветками.  
Он обеспечивает прозрачность изменений, возможность отката к предыдущим версиям и удобную совместную работу через Pull Request  


## 2. Использованные команды

- git --version
- git clone
- git status
- git add
- git commit
- git push
- git switch -c
- git pull
- git log --oneline
- git show
- git rm

## 5. Инцидент: секрет попал в репозиторий

Был создан файл secrets.txt с содержимым:
API_KEY=DEMO_LEAK_999

Файл был закоммичен и отправлен в репозиторий

### Обнаружение утечки

Утечка была обнаружена с помощью команд:

- git log --oneline -- secrets.txt
- git show HEAD:secrets.txt

### Реакция на инцидент

1. Файл secrets.txt был добавлен в .gitignore
2. Файл был удалён из репозитория через git rm
3. Изменения были закоммичены и отправлены в GitHub

### Почему удаления недостаточно

Удаление файла из текущей версии репозитория не удаляет его из истории коммитов 
Секрет остаётся доступным в старых версиях проекта

### Что необходимо сделать в реальном проекте

- Немедленно выполнить ротацию секрета
- Очистить историю репозитория с помощью инструментов
